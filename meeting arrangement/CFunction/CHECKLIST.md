# 部署检查清单

## 文件完整性检查

部署前请确认以下文件存在且完整：

- [ ] `scf_index.js` - 主入口文件（约 8-9KB）
- [ ] `scf_bootstrap` - 启动脚本（172 字节）
- [ ] `package.json` - 依赖配置
- [ ] `node_modules/` - 依赖包目录（包含 mongodb 等）

## 环境变量检查（如需要）

如果 MongoDB 地址不是默认的 `49.235.189.246:27017`，请配置：

- [ ] `MONGO_URI` - MongoDB 连接地址
- [ ] `DB_NAME` - 默认 GHS
- [ ] `GHA_DB_NAME` - 默认 GHA

## 部署步骤

1. [ ] 登录腾讯云 SCF 控制台
2. [ ] 创建/选择函数 `meeting-booking-api`
3. [ ] 上传 CFunction 文件夹
4. [ ] 确认执行方法为 `scf_index.handler`
5. [ ] （可选）配置环境变量
6. [ ] 保存并测试

## 部署后验证

- [ ] 访问 `/api/test` 返回成功
- [ ] 访问 `/api/teachers` 返回教师列表
- [ ] 前端页面可以正常搜索学生

## 常见问题

**Q: 上传后提示找不到模块？**
A: 确保 `node_modules` 目录已包含在上传的文件夹中。

**Q: 数据库连接失败？**
A: 检查 MongoDB 地址是否正确，网络是否连通。

**Q: 前端无法访问API？**
A: 检查云函数是否开启公网访问，CORS配置是否正确。
